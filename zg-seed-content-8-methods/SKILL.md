# 种草力 8 术 Lite

把种草理解为“持续展示真实价值，让合适的用户看到改变的可能性”，不要把它写成硬卖、夸大或操控。

## 先判断任务

- **生成**：从零写一条内容。
- **改写**：保留用户事实与核心立场，重组为一个明确术式。
- **诊断**：指出当前内容使用了什么术式、主要问题和最优修改方向；用户没有要求改稿时，不擅自输出整篇新稿。
- **内容布局**：设计一组内容时，先做基础布局，再让不同术式交替出现。

## 工作流程

1. 提取五项输入：目标用户、产品或主张、这条内容的单一目标、真实证据、希望读者采取的下一步。
2. 信息不全时，优先用占位符保留事实空位；只有缺失会实质改变方向时才提一个关键问题。不得自行补造数字、客户原话、成交记录、限额或权威背书。
3. 从八术中只选一种主术式：
   - 问题：让用户看见一个未被说清的问题。
   - 结果：用可核实案例展示改变可能。
   - 氛围：用真实发生的参与和行动降低决策不确定性。
   - 身份：点明“这是谁的问题”，吸引同类并筛选非目标用户。
   - 反常识：纠正一个妨碍行动的旧认知。
   - 体系：证明结果来自可重复的方法，而非偶然。
   - 卡点：说出用户想做却做不到的深层阻碍。
   - 损失：展示继续不行动的具体机会成本。
4. 按“钩子—场景/证据—转折/洞察—价值落点—轻行动”组织内容。不是每条都必须有销售 CTA；有时让用户回复、思考或观察即可。
5. 删除空泛形容词、自夸、重复结论和与主术式无关的段落。口语化表达，一段只承担一个功能。
6. 做真实性与压迫感检查后再交付。

## 选择术式

| 已有材料或目标 | 优先术式 |
|---|---|
| 用户尚未意识到问题 | 问题 |
| 有真实案例、前后变化 | 结果 |
| 有真实报名、咨询、共同行动 | 氛围 |
| 用户群体边界不清 | 身份 |
| 市场被错误认知卡住 | 反常识 |
| 需要证明方法可复制 | 体系 |
| 用户懂道理但做不到 | 卡点 |
| 用户长期拖延且代价明确 | 损失 |

若两种都适合，选择证据最充分、与目标最近的一种；不要强行混合。

## 默认交付

生成或改写时输出：

1. **术式**：名称 + 一句话选择理由。
2. **成稿**：一条可直接使用的中文内容。
3. **待核实项**：仅列稿中仍需用户补充的事实占位符；没有则省略。

诊断时输出：

1. 当前术式与单一目标。
2. 最影响效果的 1–3 个问题，并引用对应短句。
3. 修改方向和一个局部示例。

## 质量门槛

- 一条内容只打透一个点，主术式必须能一句话说清。
- 任何数字、案例、用户证言、稀缺性、身份和权威信息都必须来自用户材料或可核实来源。
- 不伪造“报名 +1”“仅剩名额”“很多人都在问”等社会证明。
- 不用羞辱、恐吓、歧视或不切实际的收益承诺推动成交。
- 若产品涉及医疗、法律、投资等高风险领域，使用审慎表述并提醒用户核验合规要求。
- 保留用户本人的语气；不要默认写成夸张“爆款体”。
