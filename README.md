# ZG Skills

> 曾哥开发的 AI 时代一人公司 Skills：把获客、营销、运营、管理、产品与交付等工作，逐步沉淀为可调用、可复用的 AI 员工能力。

一个人公司不等于一个人硬扛所有部门，而是让 AI 成为不同岗位上的专业员工。ZG Skills 会围绕一人公司的真实工作流持续开源：你给出业务处境、材料和目标，Agent 通过对应 skill 帮你做判断、产出内容，并推进下一步行动。

## 已发布 Skills

| 真实处境 | 使用 Skill | 你会得到 |
|---|---|---|
| 听到反对意见就急着解释；想表达又不敢；学习、关系或管理总是卡住 | `$zg-openness-degree` | 卡点判断、低风险行动和可复盘的问题 |
| 不知道朋友圈该发什么；内容太硬、没人回应；想做私域种草但不想夸大 | `$zg-seed-content-8-methods` | 合适的内容术式、可发布文案或针对原稿的修改方向 |

## 开源路线图：一人公司的 AI 员工

ZG Skills 将持续围绕一人公司的各个“部门”开源。以下是当前方向，不代表已经发布或承诺具体上线时间。

| 部门 | 计划开源的 AI 员工能力 |
|---|---|
| 获客与营销 | AI 做短视频与公众号获客、朋友圈营销种草、一对一私聊 |
| 运营与管理 | 招聘、企业战略管理与赛道选择、产品打磨、交付 |
| 创始人能力 | 学习、沟通、关系、决策与行动卡点 |

每个方向都会尽量沉淀为独立的 ZG Skill：有明确的适用场景、输入要求、工作流程、输出格式和边界，而不只是一个泛泛的提示词。

## 30 秒开始

### 1. 安装

把需要的 skill 复制到 Codex 的 skills 目录：

```bash
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R zg-openness-degree "${CODEX_HOME:-$HOME/.codex}/skills/"
cp -R zg-seed-content-8-methods "${CODEX_HOME:-$HOME/.codex}/skills/"
```

刷新或重启 Codex 后即可使用。只安装当前需要的 skill 即可。

### 2. 直接说出你的真实问题

当前已发布的 skill 覆盖“创始人自身”与“朋友圈/私域内容”。不需要先掌握方法论，直接说出你的真实问题即可。

```text
$zg-openness-degree
我每次开会被反对就急着解释，事后又觉得自己没听进去。帮我看看卡在哪里，并给我一个今天就能做的练习。
```

```text
$zg-seed-content-8-methods
我做创业者私域咨询，想发一条朋友圈：很多人不是没流量，而是没有持续经营已有用户。没有案例和数据，不要太营销。
```

## 已发布能力详解

### ZG · 打开度

`$zg-openness-degree`

用“打开度”框架分析学习、沟通、关系、管理、亲子与情绪中的封闭信号。它不会给人贴“你就是封闭”的标签，而是帮你区分必要边界与惯性防御，找到一个低风险、可撤回的打开动作。

适合这样问：

- “我总听不进不同意见，到底在防御什么？”
- “我知道该主动，但就是不敢联系别人，怎么办？”
- “孩子不愿意和我沟通，我可以先调整什么？”

常见输出：场景还原 → 主卡点 → 保护与代价 → 最小行动 → 复盘问题。

### ZG · 种草力 8 术 Lite

`$zg-seed-content-8-methods`

用于生成、改写或诊断朋友圈、私域和个人品牌内容。它会从问题、结果、氛围、身份、反常识、体系、卡点、损失中选择一个主术式，确保一条内容只打透一个点。

适合这样问：

- “帮我把这段介绍改成不硬卖的朋友圈。”
- “我有用户反馈，怎么写成真实、有说服力的案例内容？”
- “这条朋友圈为什么显得很营销？帮我诊断，不要直接重写。”

常见输出：术式选择理由 → 可发布成稿或诊断建议 → 待核实的事实项。

## 使用原则

- 先给真实处境、真实材料和真实目标；信息不全时，skill 会保留事实空位，而不是编造细节。
- `zg-openness-degree` 不提供医学或心理疾病诊断，也不鼓励无边界地暴露自己。
- `zg-seed-content-8-methods` 不捏造案例、数据、证言、稀缺性或身份背书；不靠恐吓和夸大推动成交。
- 你可以在同一段对话中继续补充事实、反馈和新材料，让 Agent 把下一步往前推进。

## 项目结构

```text
ZG Skills/
├── zg-openness-degree/            # 打开度：个人成长、关系与行动卡点
│   ├── SKILL.md
│   ├── agents/openai.yaml
│   └── references/
└── zg-seed-content-8-methods/     # 种草力 8 术：内容与私域表达
    ├── SKILL.md
    ├── agents/openai.yaml
    └── references/
```

每个子目录都是一个可独立安装的标准 skill。主 `SKILL.md` 只保留触发条件、工作流和边界；详细框架与模板按需放在 `references/`，避免每次调用加载不必要的内容。
